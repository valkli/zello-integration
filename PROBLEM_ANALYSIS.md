# Детальный анализ проблемы декодирования Opus аудио от Zello Work

## Суть проблемы

Zello Work отправляет Opus аудио в **кастомном бинарном формате**, который не является стандартным Ogg Opus контейнером или сырыми Opus пакетами. Все попытки декодирования этих данных не увенчались успехом.

## Формат данных Zello

### Структура пакетов

Zello отправляет бинарные пакеты через WebSocket. Наблюдаемые паттерны:

1. **Паттерн 1**: `01 00 00 2c 96 00 00 00 00 18 e1 [opus_data...]`
   - Первые 4 байта: `01 00 00 2c` - возможно, тип пакета и длина
   - Байты 4-8: `96 00 00 00` - возможно, дополнительные заголовки
   - Байты 8-10: `00 18` - маркер?
   - Байт 10: `e1` - часть маркера?
   - После этого должны идти Opus данные

2. **Паттерн 2**: `27 8f 19 96 7b 69 8f e4 60 7d 9f a6 ca 01 2c 08 d8 08 9a 90...`
   - Данные начинаются сразу с похожих на Opus байтов
   - Но TOC байт `0x27` не является валидным Opus TOC (config должен быть 0-7, но `0x27 & 0x07 = 7` - валидно, но frame count `(0x27 >> 4) & 0x07 = 2` - тоже валидно)

3. **Codec Header**: `gD4BPA==` (base64) = `803e013c` (hex) = 4 байта
   - Это заголовок кодека, который Zello отправляет в команде `on_stream_start`

### Что мы знаем из логов

- **packet_duration**: 60 (миллисекунд?)
- **codec**: "opus"
- **codec_header**: 4 байта (base64)
- Размер пакетов: 60-180 байт
- Общий размер потока: 2000-5000 байт для короткого сообщения

## Что мы пробовали

### 1. Извлечение Opus данных из пакетов

**Методы:**
- Поиск паттерна `18 e1`, `18 c1`, `18 27`, `18 15`
- Пропуск фиксированных заголовков (4, 8, 12, 16 байт)
- Валидация TOC байта для проверки валидности Opus пакета

**Результат:** Данные извлекаются, но не декодируются.

### 2. Декодирование через ffmpeg

**Попытки:**
- Разные входные форматы: `opus`, `libopus`, `ogg`
- Разные смещения (offsets): 0, 4, 8, 12, 16, 20, 24 байта
- Прямое декодирование сырых данных

**Результат:** Все попытки неудачны. ffmpeg выдает ошибки:
- "Invalid data found when processing input"
- "Error opening input"

### 3. Создание Ogg контейнера

**Методы:**
- Ручное создание Ogg страниц с правильными checksums
- Использование `opusenc` (не найден в системе)
- Использование `ffmpeg` для конвертации сырых Opus в Ogg

**Результат:** Ogg контейнер создается, но:
- `ffmpeg` не может его декодировать
- `pyogg.OpusFile` не может его прочитать
- Файл не является валидным Ogg Opus файлом

### 4. Использование библиотек

**opuslib:**
- Не может найти библиотеку Opus (требует установки libopus)
- `Exception: Could not find Opus library. Make sure it is installed.`

**pyogg:**
- Установлен, но:
  - `OpusDecoder` требует сложной инициализации через `opus_decoder_create` с параметром `error`
  - `OpusFile` требует валидный Ogg файл, который мы не можем создать

**opusdec (opus-tools):**
- Не установлен в системе
- Не доступен через pip

## Почему не работает

### Основная проблема

**Zello использует проприетарный формат упаковки Opus пакетов**, который:
1. Не является стандартным Ogg Opus контейнером
2. Не является последовательностью сырых Opus пакетов
3. Содержит кастомные заголовки, которые мы не можем правильно распарсить

### Конкретные проблемы

1. **Неизвестная структура заголовков:**
   - Мы не знаем точную структуру заголовков Zello
   - Не знаем, как правильно извлечь Opus пакеты
   - Не знаем, есть ли дополнительные метаданные

2. **Формат Opus пакетов:**
   - Даже после извлечения данных, они не являются валидными Opus пакетами
   - Возможно, Zello использует кастомную упаковку Opus пакетов
   - Возможно, нужна дополнительная обработка данных

3. **Отсутствие документации:**
   - Нет официальной документации по формату аудио Zello Work API
   - Нет примеров декодирования аудио от Zello
